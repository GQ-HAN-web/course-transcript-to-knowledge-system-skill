# 课程转写 → 知识体系提炼流程

把口语课程转写整理成可对外的教法/知识体系或书稿的标准作业流程。本技能的价值不在任何具体概念，而在这套**可复用的方法 + 防翻车的质检关卡**。

## 核心定位：带质检关卡的半自动流程

这**不是**一键生成工具。它把真实项目中趟出来的人工关卡固化下来。批处理重活（切片、编码、统计）交给执行器（Codex / Claude Code 等）跑；判断类、决定全局走向的环节由人把关。三道验收门是**硬性停止点**——执行到门口必须停下等人确认，不许自行冲过。

这样设计是因为：实践证明，正是这些人工关卡拦住了"词典匹配冒充开放编码""切片过碎导致证据全是半句话""师生角色误判"等会污染整个体系的错误。省掉关卡图快，下一个项目必然重蹈覆辙。

## 何时用本技能

适用：用户有成批口语转写（课程、讲座、答疑、研讨），要整理成概念体系 / 学习路径 / 书稿 / 长期可查的知识库。

不适用：单篇总结、翻译、普通摘要、几段文字的整理——这些直接做，不要套这套重流程。

## 启动一个新项目：第一步永远是填配置表

每个新项目第一件事是复制 `templates/项目配置表_模板.md` 并填写。配置表分离了"每次都一样的流程"和"每个项目不同的内容"：

每次不同、必须重填的：课程名、转写文件路径、分卷方式、本课程的双链概念种子词表（重新统计）、演化轴数据（重新跑）、A–H 功能模块（按本课程核心概念微调）、书稿目录骨架、以及（若是进阶/续课）前序课程的体系产物路径。

配置表没填好之前，不要进入 P1。

## 九阶段总流程

| 阶段 | 内容 | 分工 | 关卡 |
|---|---|---|---|
| **P0** | 填项目配置表 | 人填 | |
| **P1** | 论证单元切片 | 可托管 | |
| **P2** | 开放编码（行为+概念+原话锚点） | 设标准后托管 | |
| **P3** | 双链漏检校验 | 可托管 | 🚪 门一 |
| **P4** | 概念归并 + 功能定位 | 人亲自定夺 | 🚪 门二 |
| **P5** | 概念卡批量填写 | 审样卡后托管 | |
| **P6** | 演化追踪（是否修正） | 人亲自定夺 | |
| **P7** | 岔路谱（误区→校正） | 人亲自定夺 | 🚪 门三 |
| **P8** | 成书组织 + 样章 | 把关后托管 | |
| **P9** | 跨课程衔接（续课专属） | 人亲自定夺 | |

### 分工三类

- **可托管（绿）**：规则明确、可回归、错了显眼。P1 切片、P3 双链校验。机器跑，人抽查。
- **设标准后托管（黄）**：机器干活，但人先用样例定标准/定调，再批量跑，人抽查。P2 编码、P5 填卡、P8 起草。
- **人亲自定夺（红）**：判断类、决定全局、错了层层传导。机器只供料，结论人下。P4 归并、P6 演化判断、P7 岔路根源、P9 衔接。

### 三道验收门（硬性停止点）

- **门一（P3 后）**：切片 + 编码无遗漏，才进概念归并。核心校验：去重概念清单比对双链种子词表，确认高频概念无漏检。
- **门二（P4 后）**：概念骨架人确认，才批量填卡。
- **门三（P8 前）**：岔路谱 + 演化表人核完，才动笔成书。

执行到门口，停下，向人汇报本阶段产出 + 待确认项，等明确放行再继续。

## 每个阶段怎么做

每个阶段的执行细节、提示词、验收要点分别在 references 里。**按阶段按需读取，不要一次性全读**：

- 切片规范、开放编码三步、字段定义 → `references/01_编码手册.md`
- 概念卡 / 岔路谱 / 演化表 / 书稿目录模板 → `references/02_模板库.md`
- P1–P9 每步给执行器的提示词（执行器无关，带占位符）→ `references/03_提示词库.md`
- 每步验收时人/Claude 要检查什么（防词典匹配等）→ `references/04_监察清单.md`
- 真实踩坑案例（务必在 P1/P2 开始前读）→ `references/05_防坑案例.md`
- 课程体系标准产物格式（P8 产出、P9 读取）→ `references/06_产物格式规范.md`

## 方法论要点（为什么这么设计）

1. **真开放编码，不是词典匹配**：概念必须从文本自下而上长出，严禁先设概念清单再全库匹配。这是头号翻车点，P2 必须用"去重概念清单含不含冷僻词"来校验。
2. **词频 ≠ 重要性**：口语教法里最核心的常说得最少（如某些底层洞见），高频词多是教学黏合剂。用结构地位排座次，频率仅作辅助。
3. **抓住时间轴**：成批连续语料的独特价值是能看教法演化。务必跑演化轴（早期讲法 vs 后期讲法、是否修正），别拍平成词频快照。
4. **用作者已标注的概念**：若转写带双链 `[[ ]]` 或作者标签，那是最权威的概念边界，作为种子词表用于漏检。
5. **保留"反面"**：教法体系里"岔路/误区/陷阱"与"正路"同等重要，单独成谱（P7），别混进答疑按文件类型分。
6. **切片按论证单元**：以"讲完一个完整意思"为边界，不按说话人时间戳切，否则证据全是半句话。

## 跨课程衔接（P9，续课专属）

若本项目是某课程的进阶/续课，P9 对照前序课程的体系产物，标出每个概念是**延续 / 深化 / 全新引入**，使多本书稿形成有层次的整体。前提：前序课程产物须为 `references/06_产物格式规范.md` 规定的标准格式（走"格式先立好"路线）。P9 细节见提示词库与产物格式规范。

## 全程硬约束

1. 不把转写逐字拼贴成书；重复内容合并归纳。
2. 不凭空补素材里没有的故事/观点/结论。
3. 可润色口语、可适度提炼改写表达密度，但**不改教理/原意立场**，不替作者"学术化纠偏"，不自造概念。
4. 同一概念的不同说法整理成"定义 + 使用场景 + 常见误区 + 练习方法"。
5. 不确定就列入待确认，不编造人名/日期/因果/术语含义。
6. 每个核心观点都能回到源文件#段。

## 跨会话交接

项目周期长、跨多个会话时，用 `templates/交接包_模板.md` 打包当前进度、已定决策、待办，新会话贴入即可无缝继续。
