# 课程体系标准产物格式规范

> P8 产出、P9 读取的标准格式。走"格式先立好"路线（方案 A）：每门课的体系产物都整理成本规范，续课的 P9 才能直接读取比对。
> 即使当前项目不是续课，也按本规范产出——为将来的进阶/续课留好接口。

## 为什么要标准格式
跨课程衔接（P9）要对照前序课程，标出每个概念是延续/深化/新引入。若两门课各用各的格式，没法比对。统一格式让多门课的体系形成有层次的整体。

## 标准产物清单
每门课完成后，产出目录应含：

```
{{课程名}}_知识体系/
├── 概念清单.json          ★ P9 主要读取对象
├── 概念卡/                每概念一卡（Markdown）
├── 岔路谱.md
├── 演化追踪表.md
├── 书稿/
├── 证据索引.json
└── 待人工确认清单.md
```

## 概念清单.json 格式（P9 核心接口）

```json
[
  {
    "概念名": "觉知",
    "功能位": "A",
    "一句话定义": "……",
    "别名": ["看见", "只是看见"],
    "总频次": 2583,
    "分卷频次": {"P01": 908, "P02": 621, "P03": 435, "P04": 619},
    "演化趋势": "下降线/上升线/平稳",
    "代表原话锚点": ["看只是看", "……"],
    "证据切片": ["slice_id_1", "slice_id_2"],
    "概念卡路径": "概念卡/觉知.md"
  }
]
```

字段说明：
- `概念名`：归一后的标准名（P4 定稿）。
- `功能位`：A–H 或"贯穿性"。
- `别名`：归并进来的同义/近义表达，供 P9 跨课程匹配（续课可能用别名指同一概念）。
- `分卷频次` + `演化趋势`：P9 判断"深化"的依据（续课频次更高/讲法更深=深化）。
- `代表原话锚点`：供 P9 比对讲法是否一致（延续=讲法基本一致）。

## P9 输出格式（衔接对照表）

```json
[
  {
    "本课程概念": "无我",
    "衔接类型": "深化",
    "前序对应概念": "无我",
    "判断依据": "前序课程已引入但频次低、停在概念层；本课程频次升高且与行愿/利他绑定，推进到行动层",
    "本课程证据": ["slice_id"],
    "前序证据": "前序概念卡路径"
  }
]
```

`衔接类型` 三值：
- **延续**：前序已有，讲法基本一致。
- **深化**：前序已有，本课程往更深推进（频次升高 / 与更多概念绑定 / 从概念层到行动层）。
- **新引入**：前序没有。

判定由人确认，机器只供对照和证据。

## 衔接成书的价值
P9 产出让续课书稿能写出"本阶段在前一阶段基础上往哪里推进了"，形成有层次的整体，而非孤立的两本书。这正是续课项目相对单课项目的增值点。
