# 监察清单（每步验收查什么）

> 给把关人 / Claude 用：每个阶段执行器交回产物后，按本清单逐项核。打勾才放行。

## P1 切片验收
- [ ] 每分卷抽查 3–5 片：完整论证有没有被切断（证据是否半句话）
- [ ] 角色判定：自述体验有没有被误判为讲者讲法（反之亦然）
- [ ] 现场杂音有没有混进"教学"（尤其"大家轮流分享"类组织过场）
- [ ] 待确认标记有没有漏（括号不配对、错字、人名不一致）
- [ ] 切片类型分布是否合理（杂音占比过高=切太碎；教学占比异常=角色判错）

## P2a 行为编码验收
- [ ] 多标签是否合理（讲者段常该多标）
- [ ] 师/学员标签有没有混用

## P2b 概念编码验收（门一前核心，防词典匹配）
- [ ] **去重清单里有没有冷僻概念**（绝不可能在预设词典里的词）→ 有=真开放编码；只有几个高频"老面孔"=套词典，打回
- [ ] 抽一段概念密集的教学切片，核对动词性/否定性表达有没有被抽到（"超越二元""不再选择""如实"最易漏）
- [ ] 颗粒度是否一致（不能一片 15 个、另一片更丰富却 2 个）
- [ ] 概念标签有没有混入叙事名词或整句（应只在原话锚点里）

## 🚪 门一（P3 后）
- [ ] 去重概念清单已比对双链种子词表，高频概念无漏检
- [ ] 重点核对的概念（配置表指定）全部覆盖
- [ ] 分卷频次表方向与预期演化一致
- [ ] 全部通过 → 放行进 P4

## P4 概念归并验收（人定夺，不托管）
- [ ] 机器只给了候选分组，没有擅自定最终归并
- [ ] 归并颗粒度由人拍板（近义概念合一还是分立）
- [ ] 每个核心概念都定了功能位
- [ ] 被漏的高频概念（如某些贯穿性概念）有没有进正式清单

## 🚪 门二（P4 后）
- [ ] 概念骨架人已确认 → 放行批量填卡

## P5 概念卡验收
- [ ] 样卡：定义是否立得住（不是抄原文片段）
- [ ] 改写有没有越界改了原意/教理立场
- [ ] 原话引用是否逐字、标源、碎段未引、每概念≤3条

## P6 演化追踪验收（人定夺）
- [ ] 机器只并列摆出早晚期原话，没擅自下"是否修正"结论
- [ ] 人逐条判断：加深 / 修正 / 仅换措辞（别把换说法误判为修正立场）

## P7 岔路谱验收（人定夺）
- [ ] 机器只给"误区—校正原话对"，没擅自写根源
- [ ] 人补根源诊断 + 正路指向

## 🚪 门三（P8 前）
- [ ] 岔路谱 + 演化表人已核完 → 放行动笔成书

## P8 成书验收
- [ ] 改写尺度：有没有口水化或学术化失真
- [ ] 教理/原意忠实性
- [ ] 六条硬约束逐条核
- [ ] 待人工确认清单是否产出

## P9 跨课程衔接验收（续课，人定夺）
- [ ] 前序产物是标准格式
- [ ] 机器只做对照供料，延续/深化/新引入最终判定由人确认
- [ ] 每条对照都有双向证据（本课程切片 + 前序概念卡）
