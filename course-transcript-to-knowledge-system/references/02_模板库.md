## 概念卡模板（P5）

```
## 概念名（功能位 X）

**一句话定义**：（书稿语言，立得住，可适度改写表达密度，不改原意）

**在体系中的位置**：上承___，下启___，与___相邻，与___易混。

**边界辨析**：它不是___（区分常被混淆的近义概念）。

**讲法演化**（来自 P6 演化轴）：
- 早期：讲者怎么说……
- 后期：讲法是否变化 / 加深 / 被修正……
- 是否 supersedes 早期某说法：是/否，证据。

**常见误区 → 校正**（链入 P7 岔路谱）：
- 误区：……｜校正：……｜证据：源文件#段

**练习提示**：（如有）

**高价值原话**（≤本概念 2–3 条，逐字，标源文件，碎段不引）：
> "……"  ——源文件 日期

**关系**（概念图边）：derives-from / supports / blocks / supersedes / commits-to
```

填写要点：定义要立得住（不是抄原文片段）；原话引用逐字、标源、碎段不引、每概念 ≤3 条。

## 岔路谱模板（P7）

```
## 岔路：误区名

**表现**：学员典型说法 / 状态。
**根源**：为什么会掉进去（此栏由人诊断，机器不写）。
**讲者如何校正**：原话脉络 + 提炼。
**正路指向**：应回到哪个概念 / 练习。
**证据**：源文件#段（优先用答疑、个案）。
```

岔路谱是教法体系最精彩的部分，通常成书时单独成章。机器只供"误区—校正原话对"，根源诊断和正路指向由人补。

## 演化追踪表模板（P6）

```
| 概念 | 早期讲法 | 后期讲法 | 是否修正/加深/仅换措辞 | 关键转折文件 |
|------|---------|---------|----------------------|-------------|
```

判断要点：机器只并列摆出早晚期原话，**不下"是否修正"结论**；人逐条判断是加深、修正、还是仅换措辞。把"换个说法"误判成"修正立场"是常见错误。

## 书稿目录骨架模板（P8）

按读者**进阶次第**组织，不按文件类型（答疑/结营）分章。双线并行：明线是次第（入口→障碍→转化→洞见→方向），暗线是演化（早期重心 → 后期重心）。

```
序 · 本课程的核心入口

第一部 · ［入口能力，重心：早期语料］
第二部 · ［核心障碍 + 岔路专章 ← P7 整章］
第三部 · ［转化之道］
第四部 · ［底层洞见］
第五部 · ［方向落点，重心：后期语料，演化的落点］
第六部 · ［教与学：答疑现场的生命力］

附录：概念词典 ｜ 演化追踪表 ｜ 岔路谱总表 ｜ 证据索引 ｜ 待人工确认清单
```

每章配素材来源表（本章用到哪些概念卡 / 切片）。建议先写"岔路专章"作样章——最能体现差异化价值。

## A–H 功能模块（P4 归并维度）

这是概念归并时的功能定位维度。下列为参考骨架，**每个项目按本课程核心概念微调**（进阶班的模块可能与扎根班不同）：

- **A 入口能力**：把人从自动反应带出来的基础能力
- **B 被看见的对象**：练习中辨认的当下经验对象
- **C 转化方式**：从对抗转为接纳/慈悲的路径
- **D 核心障碍**：遮蔽与陷阱
- **E 底层洞见**：实相层面的洞见
- **F 实践路径**：把理解落到身体与生活
- **G 教学机制**：答疑、个案、阶段转化
- **H 方向/行动**：把洞见导向生命方向与行动
- **贯穿性**：跨模块的概念（如"智慧"这类），单列

归并时为每个核心概念定一个功能位。颗粒度（两个近义概念合一还是分立）由人定夺，机器只给候选分组。
