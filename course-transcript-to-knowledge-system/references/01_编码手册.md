## P1 · 论证单元切片规范

### 切分单位
单位 = **完整论证单元**，不是定长，不按说话人时间戳切。
一个单元 = 讲者围绕一个点把话说完（含铺垫、打比方、落点），通常 200–800 字。

### 切分规则
1. **跨时间戳合并**：讲者连续讲述时，即使中间被听众简短附和/回应打断，也合为一个切片。听众的实质分享或提问各自独立成片。
2. **切分落点信号**：话题转换、"好我们说下一个"、从讲理论转入布置练习、听众换人提问。不要一换说话人就断。
3. 宁可完整勿凑长度。一段完整论证 900 字也不拆；一句过场 20 字也不并。

### 角色判定（关键）
**以内容口吻为准，不以说话人时间戳位置为准。**
- 讲者特征：对人讲法、印证、提点、下断语（"你本来就是…""这就是…""你要去…"）
- 听众特征：第一人称自述体验或提问（"我这两周…""我的问题是…""我感受到…"）
- 注意：听众分享中常出现自我对话式的第二人称（"你越松弛越敞开"其实是学员在描述自己），不要误判为讲者讲法。

### 切片类型
`教学`（讲者讲法）/ `学员分享` / `学员提问` / `师生问答` / `现场杂音`
- **现场杂音**：设备调试、点名、课堂组织过场（如"大家轮流分享一下""从你这边开始"）。标为杂音的**不进后续编码**，但保留以备追溯。
- 课堂组织过场是杂音，不是"布置练习"。布置练习是给具体修行/操作指导。

### 数据质量标记
凡括号/标记不配对、明显错字、人名不一致、疑似转写错误：`需人工确认 = true`，并在 `确认点` 写明问题。**不自行改原文。**

### P1 输出字段
`slice_id, 源文件, 日期, 分卷, 说话人角色(师/学员), 课程类型, 切片类型, 需人工确认, 确认点, 正文`

### P1 验收（人做）
每个分卷抽查 3–5 片：切片有没有把完整论证切断、角色判定对不对、杂音有没有混进教学、待确认有没有漏标。通过后才放行全量。

## P2a · 行为编码

对每个非杂音切片打 1–N 个行为标签（可多选，讲者常先纠错再打比方再布置练习）。

### 标签清单
讲者专用：`下定义 / 纠错校正 / 打比方 / 讲亲身经历 / 布置练习 / 回应学员困惑 / 立方向 / 破执着 / 反问 / 印证`
听众专用：`学员自述体验 / 学员提问`

### 规则
- 一切片可多标签。
- 听众切片只用听众标签，不混用讲者标签。
- 输出新增字段 `行为标签: [...]`。

## P2b · 概念编码（头号翻车点，必须真开放编码）

### 必须做
1. 用**原文词**，不归并、不翻译、不学术化。原文说"着相"就写"着相"，不替换成"执着于表象"；原文说"化掉"就写"化掉"，不改成"消解"。
2. 双链 `[[ ]]` 或作者标注里的词必须抽出。
3. 没有标注但明显是核心表达的也要抽（如"修行的执着""超越二元""如实如是""看只是看"）。
4. **动词性表达**（"看见""放下""超越""相续""扎根"）和**否定性表达**（"不是不要""不再选择"）同样重要——最容易漏，也最是教法精华。
5. 每切片抽 0–N 个（纯过场或纯情绪表达可以是 0）。

### 严禁做
1. **不要预设任何概念清单再去匹配**——概念必须从正文逐片长出。
2. 同义词不归并（"慈爱"和"慈悲"分开，"着相"和"我执"分开）。归并是 P4 的事。
3. 不自造原文没有的概念。

### 概念标签的粒度（排除两类噪声）
1. **排除纯叙事名词**：与教法无关、只描述听众经历背景的词（"线下课""听音频""走路""报名""某某营"）不进概念标签。
2. **排除整句/整短语**：不要把一整句当一个标签。"愿我化作光明雨露"→拆出"化作""光明""雨露"；"如是如是如是"→"如是"（去字面重复）。

### 原话锚点（单独字段）
把值得做书中引文/金句的**完整表达**存进 `原话锚点`（如"看只是看""你的存在就是在滋养着一切"）。此字段不参与 P3/P4 概念比对，只作书稿引用库备用。

### P2 输出字段（在 P1 基础上新增）
`行为标签: [...], 概念标签: [...], 原话锚点: [...]`

### P2 验收（人做，门一前的核心）
1. 抽查样例：看它"怎么抽"。
2. 看去重概念清单：**有没有出现冷僻、绝不可能在预设词典里的概念**（如"调平""玩乐的心态""开放式的态度"）。有→是真开放编码；只有"老面孔"几个高频词→在套词典，打回重做。
3. 看颗粒度是否一致：不能一片抽 15 个、另一片更丰富却只抽 2 个。

## 字段总表

| slice_id | P1 | 唯一编号，建议含分卷+日期+源文件+序号 |

| 日期 | P1 | YYYY-MM-DD |
